# MVP PRD

## 1. 文档说明

- **版本**：MVP（P0）
- **目标**：定义产品 MVP 核心功能范围与验收标准

---

## 2. MVP 范围概览

| 模块           | 名称       | 优先级 |
|----------------|------------|--------|
| 4.1            | 用户建档   | P0     |
| 4.2            | 录入灵感   | P0     |

---

## 3. 用户建档（Creator Profile）

### 3.1 目的

获取足够的创作者画像，用于后续拆解偏好、推荐排序，以及生成建议时的语气/风格定制。要求「填得快」，但要能支撑个性化。

### 3.2 必填项

| 字段               | 类型     | 说明 |
|--------------------|----------|------|
| 内容领域           | 多选     | AI工具 / 终身学习 / 读书 / 生活方式 / 其他（可自定义） |
| 主要平台偏好       | 多选     | YouTube / 小红书 / B站（作为提示语与推荐权重，不影响功能可用性） |
| 输出形式           | 多选     | 横版口播 / 竖版短视频 / 图文 |
| 博主类型与内容描述 | 短文本   | 1–3 句。示例：我做 AI 工具与学习方法内容，偏干货、结构清晰、标题要有冲突感但不夸张；目标是增长 YouTube 订阅。 |

### 3.3 选填项

| 字段               | 类型     | 说明 |
|--------------------|----------|------|
| 风格关键词         | 选择 3–5 个 | 克制 / 犀利 / 温柔 / 学术 / 幽默 / 故事化 / 高能快节奏 / 生活化 / 反差感 |
| YouTube Channel 链接 | 可选     | 用于轻量画像分析。MVP 仅做「频道级快速分析」：基于频道公开信息（频道简介/名称/公开封面等）+ 用户自述，生成：内容定位、受众猜测、表达风格建议。**不做**：抓取全量历史视频、逐条分析、频道增长数据深挖。 |

### 3.4 输出（Profile Summary）

系统生成一段「创作者画像摘要」（可编辑），用于后续提示语与 UI 展示：

- 领域/主题关键词
- 偏好拆解维度权重（标题 / 封面 / 结构 / 表达 / 视觉制作）
- 建议的内容策略提示（1–2 条）

---

## 4. 录入灵感（Paste URL）

### 4.1 目标

约 10 秒完成一条灵感录入；用户只做极少输入，系统自动补全并生成「可复用」卡片。

### 4.2 输入

- **YouTube URL**

### 4.3 系统自动抓取并展示（P0 必须）

| 字段       | 必选 | 说明 |
|------------|------|------|
| 标题       | ✓    | 必须展示 |
| 作者/频道名 | ✓    | 必须展示 |
| 封面图     | ✓    | 必须展示 |
| 发布时间   | ✓    | 必须展示 |
| 视频描述   | ✓    | 若可得则展示 |
| 时长       | 可选 | 若可得 |
| 观看量     | 可选 | 若可得，非必需 |

### 4.4 用户交互（极简标注）

用户只需完成：

1. **「我喜欢它哪里」**（多选，分组呈现以降低认知负担）
   - 文案/策略类：标题 / 开场钩子 / 结构 / 表达金句
   - 视觉/制作类：封面 / 转场 / 调色 / 特效
2. **（可选）一句话手写备注**
   - 示例：这个标题用「对比 + 结果」，封面文字很少但很抓人；开场第一句直接抛痛点。

> **说明**：MVP 暂不强制「我存它是为了什么」（purpose）；建议保留为 **P0.5 可选字段**，便于后续推荐。若坚持极简，可先只保留「喜欢点」，后续用 AI 自动推测 purpose。

### 4.5 保存后系统自动生成（P0）

**灵感卡片（Inspiration Card）** 需包含：

| 产出项       | 要求 |
|--------------|------|
| 亮点总结     | 优先围绕用户选择的「喜欢点」 |
| 可复用建议   | 至少覆盖用户选择的 1–2 个维度： |
| → 标题       | 标题类型 + 1 个模板 + 5 个同风格改写 |
| → 封面       | 封面结构/视觉锤描述 + 3 个可直接用的封面文案（含排版提示） |
| → 结构       | 3 段骨架（开场/主体/收束）+ 适用原因 |
| → 视觉制作   | 若用户选了转场/调色/特效：仅给「观察要点 + 可复用描述」，不做专业参数级建议 |
| embedding 向量 | 用于 Find Inspiration 推荐 |

### 4.6 降级策略（P0 必须）

| 场景         | 处理方式 |
|--------------|----------|
| 抓取失败     | 允许用户手动补标题、上传截图（thumbnail）后保存 |
| 无 transcript | 仍可做标题/封面/结构的轻量拆解（基于标题、描述、用户备注） |

---

## 5. 附录

- **P0.5 可选**：灵感录入中的「我存它是为了什么」（purpose）字段，便于后续推荐与 AI 推测。
